## Decisions Made

- **Automation-first** — no manual flying phase, mission engine is primary interface from day one
- **Skywalker X8** — best balance of pusher config, payload bay, ArduPilot community, price (~£120)
- **pymavlink only** — dronekit dropped (abandoned), MAVSDK considered but unnecessary for Phase A
- **pydantic over dataclass** — runtime validation, JSON serialization, consistent with tech stack
- **Minimal Phase A models** — BasicMission only, full Goal/ConstraintSet hierarchy deferred to Phase B
- **40% range warning threshold** — not 50%, because headwind on return effectively halves range
- **ERFC or Caterham** — not Epsom Downs (public land, horses, regulatory issues)
- **Conservation/survey as beachhead market** — lowest regulatory barrier, grant-fundable, matches platform strengths
- **DaaS model first** — drone-as-a-service for first revenue, open-source core for ecosystem
- **Build starts summer 2026** — after A-level Year 12 exams window and Investin work experience
- **Multi-scale platform family** — 4 tiers (MICRO/MINI/MEDIUM/LARGE) sharing common design philosophy
- **Ground mesh network** — as standalone comms engineering project, transferable to drone relay
